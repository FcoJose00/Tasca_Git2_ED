# Què és una Pull Request?

Una **Pull Request (PR)** és una sol·licitud per integrar canvis d’un *branch* a un altre dins d’un repositori.  
Permet que altres persones revisin el codi abans de fusionar-lo.

---

# Per a què serveix?

- Proposar canvis o noves funcionalitats.
- Revisar i validar el codi de manera col·laborativa.
- Detectar errors abans de fer el *merge*.
- Documentar discussions i decisions del projecte.
- Executar proves automàtiques (CI/CD).
- Integrar codi de manera segura i controlada.

---

# Avantatges

- ✔ Millora la qualitat del codi gràcies a les *code reviews*.
- ✔ Redueix errors i regressions.
- ✔ Facilita la col·laboració entre desenvolupadors.
- ✔ Documenta canvis i decisions.
- ✔ Permet proves automàtiques abans de fusionar.
- ✔ Controla millor el flux de treball.

---

# Desavantatges

- ✖ Pot alentir el procés si les revisions tarden.
- ✖ Afegeix burocràcia en projectes petits.
- ✖ Pot generar conflictes de fusió si es manté oberta massa temps.
- ✖ Massa PRs poden dificultar l’organització.
- ✖ Dependència de plataformes com GitHub o GitLab.
